PassengerId (승객 ID)
> 발급 조건이 무엇인가?에 따라 갈림
Survived (생존 현황)
> 이 정보가 Test에서 빠져있는데, 이 정보를 알아내는 게 과제
Pclass (1등석, 2등석, 3등석?)
> 승객 등급.
Name (고객의 이름)
> Mr. 와 Miss. 등이 들어있는가? (놀랍게도 계급과 관련이 있음)
Gender (고객의 성별)
> 너무 쉽게 단순화 가능 (남, 여)
Age (연령)
> 어느 정도 범위를 잡아서 단순화 가능
SibSp (함께 탑승한 형제 또는 배우자 수) (Sibling + Spouse)
> 혼자 탔는지 / 같이 탔는지로 단순화 가능, 같이 탔다면 몇 명 인지로 세분화 가능
Parch (함께 탑승한 부모님 혹은 자녀 수) (Parent + Child)
> 혼자 탔는지 / 같이 탔는지로 단순화 가능, 같이 탔다면 몇 명 인지로 세분화 가능
Ticket (티켓번호)
> 필요없음
Fare (요금)
> 위의 Ticket과 겹치는 조건
Cabin (객실)
> 이 객실이 승객이 쓰던 객실인가 승객이 죽은 채 발견된 객실인가에 따라서 갈림
Embarked (부두 위치)
> 이 부두 위치에 따라서 이 승객의 재산에 대해서 어느 정도는 짐작할 수 있음.

(Case 0) -강사님 방법
embarked_nominal -> 승객이 탑승한 부두 위치를 1,2,3으로 구분함
title_nomimal -> 이름 앞에 있던 명칭으로 기혼 여부/계급 등을 1,2,3,4,5,6으로 구분함
gender_nominal -> 남성은 0 여성은 1
age_ordinal -> Age의 데이터를 바탕으로 AgeGroup으로 묶음.
기준 나이는 0, 5, 12, 18, 24, 35, 60, 60+. -0.5는 나이불명
fare_ordinal -> 승객이 낸 운임(서로 상이함)을 바탕으로 FareBand로 묶음.
1등표 기준은 30, 2등표 기준은 13, 3등표 기준은 8, 그 이하는 안 좋은 표

정확도결과는 61.62%, 채점결과 0.75

(Case 1)
PassengerId -> 발급 조건이 그냥 발급받은 순서라 가정하고 열외해야 하지만, 이걸 출력해야 해서 남겨놓음
Survived -> 반대쪽 Test의 이 여부를 알아내는 게 조건이므로, 그냥 남겨야 함
Pclass -> 1등석은 냅두고 나머지는 전부 0 처리함
Name -> Title로 바꾸고, 부유층을 전부 1로, 일반 남성과 미혼 여성은 2, 기혼 여성은 3으로 묶음
Gender -> 남자는 0, 여자는 1로 나눔
Age -> 나이불명(-0.5)-> 0, ~18 -> 1, ~35 -> 2, ~60 -> 3, 60~ -> 4으로 구분
SibSp -> 0은 0으로 맞추고, 1 이상 수치를 전부 1로 고정함
ParCh -> 0은 0으로 맞추고, 1 이상 수치를 전부 1로 고정함
Ticket -> 열외
Fare -> PClass과 조건이 겹치므로 열외
Cabin -> 객실이 위치해있던 위치라 가정하고 A부터 B갑판 -> 1 C부터 D -> 2 나머지 0
Embarked -> S와 C는 그냥 평범한 항구도시 -> 0 Q는 부자동네 -> 1
